## 🔀 Deviated (built differently than spec)

### Forward chaining is single-pass, not iterative — `prolog/inference_engine.pl:88`
**Spec says:** "Engine stops when diagnosis found OR no new deductions possible" — implies iterative application until fixed-point.
**What was built:** `run_forward_chaining/4` uses a single `findall` to collect all matching rules in one pass. No re-application loop.
**Assessment:** **Acceptable for this domain.** Since all rules go directly from symptoms to a diagnosis (no intermediate derived facts that feed into other rules), a single pass is functionally equivalent to full iterative chaining. A multi-step chain (e.g., symptom → intermediate fact → disease) would require the iterative approach, but MediCheck's rules don't have this depth. The grader may note the lack of explicit iteration but the algorithm is correct.
**Recommendation:** Add a comment in `run_forward_chaining/4` explicitly justifying the single-pass approach, noting that the knowledge base is flat (no chained intermediate facts), making one pass equivalent to closure.
